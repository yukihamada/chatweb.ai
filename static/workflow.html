0
🧠 Memory Recall
長期記憶の参照
リクエスト処理の最初に実行。過去の会話から抽出・保存した記憶を SQLite で検索し、ユーザー固有の文脈（名前・プロジェクト・好み・制約）を取得してプロンプトに注入する。
🔍キーワード抽出（正規表現）→ LIKE 検索 → 重要度 + 最終アクセス日時でソート
📌TOP6 件を 「【長期記憶】」 ブロックとしてプロンプト先頭に追加
📤SSE イベント memories_loaded でフロントに通知（ピンクバーで表示）
SQLite LIKE 重要度1-10 アクセス履歴 セッション横断
1
🧭 Planner
マルチエージェント判定
Claude Haiku がタスクを解析し「シングルエージェントで済むか」「複数エージェントの協調が必要か」を判断。複合タスクの場合はエージェント列とサブタスクへの分解プランを返す。
✅single：「Rustで関数を書いて」→ code エージェント1本に直行
🔗multi：「調査してメールで送って」→ [research → analyst → notify] の3ステッププランを生成
⚡モデル：Claude Haiku（高速・低コスト）。最大3ステップ制限
Claude Haiku JSON出力 最大3ステップ
2
🗺️ Semantic Router
エージェント選択
シングルパスの場合のみ実行。Claude Haiku が8種類のエージェントの中から最適な1つを選択。信頼度スコア（0〜1）も出力し、UI の統計に反映。
📋選択肢：research / code / qa / analyst / legal / finance / schedule / notify
📊出力例：{"agent":"code","reason":"Rust言語名が含まれる","confidence":0.97}
Claude Haiku 信頼度スコア 8エージェント
3
🔌 MCP Tools
外部ツール実行
エージェントに応じた実ツールを並列実行してデータを収集。結果は「強化メッセージ」としてプロンプトに追記されてから LLM に渡される。
🔍research：DuckDuckGo Web検索 + Wikipedia
⚙️code：GitHub リポジトリ検索
🌐qa：Playwright ヘッドレスブラウザ → スクリーンショット + テキスト抽出
💹finance：yFinance 株価・財務データ + DCFモデル
⚖️legal：法令検索 + コンプライアンスチェッカー
DuckDuckGo Wikipedia Playwright yFinance GitHub
4
⚡ Streaming LLM
リアルタイム生成
Claude Sonnet がトークン単位でストリーミング生成。SSE（Server-Sent Events）経由でフロントに1文字ずつ届き、ユーザーは生成が完了する前からリアルタイムで読める。
🔄AsyncAnthropic messages.stream() → テキストチャンクを SSE に変換
📜会話履歴（直近8ターン）+ 長期記憶 + ツール結果 + システムプロンプトを連結
🔡SSE イベント：stream_start → token×N → done
Claude Sonnet 4.6 SSE Streaming 会話履歴 8turns
5
🔄 Agentic Loop
自己評価 → 改善
生成完了後、別の Claude Haiku インスタンスが回答を 0〜10 点で採点。7点未満なら問題点リストを付けて同じエージェントに再生成を依頼し、品質を担保する。
📊評価プロンプト → JSON出力 {"score": 8, "issues": ["具体性不足"]}
🔁スコア < 7 → self_reflect() → 改善版で上書き（最大1回）
✅スコア ≥ 7 → そのまま確定。スコアを UI の品質チップに表示
Claude Haiku 評価 0-10スコア 自動改善
6
⚠️ HITL
人間承認（危険アクション）
通知エージェント（メール・Slack・LINE）は外部への実際の送信を伴うため、AIが自動実行せず必ず人間の確認を求める。承認・却下の結果は SQLite に永続化される。
📋UI にドラフト・宛先・件名を表示 → ユーザーが「承認」or「却下」
✅承認時：Resend API（メール）または Slack Webhook で実際に送信
💾全判断履歴を SQLite に保存 → 監査証跡として参照可能
Resend Email Slack Webhook SQLite 永続化
7
🧠 Memory Extract
長期記憶の蓄積
回答確定後、バックグラウンドで Claude Haiku が会話ターンを解析し、長期的に有用な情報（名前・プロジェクト・好み・制約）を抽出して SQLite に保存。次回以降の全会話で自動参照される。
🔬Haiku に「この会話から記憶すべき情報を抽出」→ JSON配列で受取
🚫重複チェック：先頭40文字で LIKE 検索 → 既存なら追加しない
📤SSE memories_stored → フロントのメモリパネルが自動更新
Claude Haiku asyncio.create_task 重複防止 重要度1-10

コアコンセプト
実装した7つの技術要素
このシステムが体現している、現代の AI エージェントシステムの主要パターン。
🗺️
Semantic Routing
LLM自身が「どのエージェントが最適か」を判断して振り分ける。ルールベースのif-elseではなく、意味的な理解で動的にルーティング。
なぜ重要か：タスクが多様になるほどルールベースは破綻する。LLMに任せれば「株価を調べてグラフ化して」のような複合意図も正確に処理できる。
🔌
MCP (Model Context Protocol)
エージェントが外部ツール・DBに接続するための標準プロトコル。ブラウザ・検索・株価API・コード実行などを統一的に呼び出す。
なぜ重要か：AIが「知識」だけでなく「行動」できるようになる。リアルタイム情報の取得・外部システムとの連携が可能になる。
⚠️
HITL (Human in the Loop)
AIが自律的に動く中で、外部への実際の影響（送信・削除・決済）が発生する前に人間の確認を挟む安全装置。
なぜ重要か：自律AIへの信頼は「止められる仕組み」があって初めて成立する。AGI時代の責任あるAI開発の基本パターン。
🔄
Agentic Loop
生成 → 自己評価 → 改善という反復サイクル。人間のレビューなしに品質を高める自律改善メカニズム。
なぜ重要か：1回の生成で完璧な回答は難しい。評価-改善ループがあることでアウトプット品質が統計的に向上する。
⚡
Token Streaming (SSE)
LLMの生成をトークン単位でリアルタイム転送。ユーザーは完了を待たず即座に読み始められる。体感レスポンス速度が劇的に向上。
なぜ重要か：5秒の完了待ちと、即座に文字が流れてくる体験の差は UX として致命的。ChatGPTが普及した理由の一つ。
🔗
A2A Delegation
Agent-to-Agent プロトコル。複合タスクを自動分解し、専門エージェントのチェーンで実行。前のエージェントの出力が次のインプットになる。
なぜ重要か：単一の汎用AIより専門AIの連携の方が高品質。人間の組織と同じ「分業 + 連携」のパラダイムをAIに適用。
🧠
Long-term Memory
Mem0インスパイア。会話から重要情報を自動抽出してSQLiteに蓄積。セッションを跨いでユーザーを「覚え続ける」パーソナライズAI。
なぜ重要か：毎回「私の名前は〜で、プロジェクトは〜」と説明するAIは実用的ではない。記憶があることで初めて「アシスタント」になれる。
